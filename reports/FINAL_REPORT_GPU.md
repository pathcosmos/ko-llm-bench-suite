## 1. Executive Summary (경영진 요약)

FRANKENSTALLM 3B v2는 LLaMA 아키텍처 기반 1.2B 파라미터 한국어 ORPO 모델이다. 본 리포트는 7개 트랙(한국어 벤치마크, 멀티턴 대화, 한국어 심화, 코드/수학, 일관성/강건성, 성능 프로파일링, 쌍대비교)에 걸쳐 **7개 비교 모델** 대비 FRANKENSTALLM v2의 객관적 위치를 평가한다. 이전 CPU-only 평가(8개 모델)와 달리 RTX 5070 Ti 16GB GPU 풀 가속 모드에서 **10개 모델**을 대상으로 재평가했다.

### 핵심 결론

| 항목 | 결과 |
|------|------|
| **종합 순위** | **10개 모델 중 최하위** (모든 양자화 수준에서) |
| **가장 강한 영역** | GPU 추론 속도 (633 tok/s decode, Q4_K_M), 길이 민감도 |
| **가장 약한 영역** | 코드/수학 (전 영역 0%), 지시 준수 (0%), 환각 탐지 (0%) |
| **Track 7 Elo** | Q4_K_M 1000.0 (초기값) vs 1위 gemma3:4b (1834.7) |
| **GPU 가속 효과** | 추론 속도 20~50배 향상. 평가 총 시간 ~19h (CPU 대비 ~40h → 절반) |

### 새로 추가된 모델 (CPU 평가 대비)

| 모델 | 파라미터 | 특이사항 |
|------|:--------:|----------|

### 권고사항

1. FRANKENSTALLM v2는 현재 상태에서 실용적 활용이 어려움 — 근본적 모델 재설계 필요
2. gemma3:4b가 종합 성능 1위로, 3~4B급 한국어 LLM이 필요한 경우 1순위 권장
3. 한국어 특화 작업에는 exaone3.5:2.4b (Track 1: 97.8%, Track 5: 0.747)가 최적
4. 속도 우선 경량 배포에는 ingu627/exaone4.0:1.2b (Track 1: 74.8%, decode 546.5 tok/s) 고려
5. v3 개발 시 파라미터 수 확대, Instruction-following 데이터 보강, 코드/수학 혼합 학습이 필수

### 3.2 FRANKENSTALLM v2 특성

- **실제 파라미터**: 1.2B (명칭은 "3B"이나 실제는 1.2B)
- **토크나이저**: SentencePiece Model (SPM) — 대부분의 비교 모델이 BPE를 사용하는 것과 차이
- **학습 방법**: ORPO (Odds Ratio Preference Optimization) — 한국어 데이터로 fine-tuning
- **v1 vs v2**: v1은 SPM 결함으로 실행 불가(CRASH), v2는 실행 가능하나 품질 미달

### 3.3 비교 모델 선정 근거

- **기존 5개 모델**: 2~4B 파라미터 범위, 한국어 지원, Ollama 추론 가능
- **llama3.1:8b**: 8B급 상한 기준선 — FRANKENSTALLM v2가 동급 1.2B 모델 대비 어느 위치인지 파악
- **ingu627/exaone4.0:1.2b**: 동일 파라미터(1.2B) 경쟁 모델 — 한국어 ORPO 학습의 상대 평가
