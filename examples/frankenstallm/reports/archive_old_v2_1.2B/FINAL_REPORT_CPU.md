# FRANKENSTALLM 3B v2 최종 평가 리포트

> **평가 기간**: 2026-03-10 ~ 2026-03-12 (총 ~40시간)
> **작성일**: 2026-03-13
> **평가 모드**: CPU-only (GPU 의도적 미사용)
> **Judge LLM**: Claude (Anthropic, `claude -p` CLI)

---

## 1. Executive Summary (경영진 요약)

FRANKENSTALLM 3B v2는 LLaMA 아키텍처 기반 1.2B 파라미터 한국어 ORPO 모델이다. 본 리포트는 7개 트랙(한국어 벤치마크, 멀티턴 대화, 한국어 심화, 코드/수학, 일관성/강건성, 성능 프로파일링, 쌍대비교)에 걸쳐 5개 비교 모델 대비 FRANKENSTALLM v2의 객관적 위치를 평가한다.

### 핵심 결론

| 항목 | 결과 |
|------|------|
| **종합 순위** | **8개 모델 중 최하위** (모든 양자화 수준에서) |
| **가장 강한 영역** | GPU 추론 속도 (633 tok/s decode, Q4_K_M), 길이 민감도 (0.6~1.0) |
| **가장 약한 영역** | 코드/수학 (전 영역 0%), 지시 준수 (0%), 환각 탐지 (0~20%) |
| **Track 7 Elo** | ~1000 (초기값 수준) vs 1위 gemma3:4b (1734) |

### 권고사항

1. FRANKENSTALLM v2는 현재 상태에서 실용적 활용이 어려움 -- 근본적 모델 재설계 필요
2. gemma3:4b가 종합 성능 1위로, 3~4B급 한국어 LLM이 필요한 경우 1순위 권장
3. 한국어 특화 작업에는 exaone3.5:2.4b (Track 1: 97.8%, Track 3: 71.3%)가 최적
4. v3 개발 시 파라미터 수 확대, Instruction-following 데이터 보강, 코드/수학 혼합 학습이 필수

### 2.2 CPU-only 테스트 결정 사유

본 평가는 **GPU(RTX 5060 Ti 16GB)가 장착된 환경에서 의도적으로 CPU-only 모드로 수행**되었다.

- `CUDA_VISIBLE_DEVICES=""` 환경변수로 GPU 비활성화
- CPU 모드로 인해 타임아웃 2배 자동 적용 (`config.py _TIMEOUT_MULTIPLIER=2`)
- Track 6 성능 프로파일링에서만 GPU 모드 참고 측정 실시

**중요**: 정확도 평가(Track 1, 4)는 greedy decoding을 사용하므로 GPU/CPU 결과가 이론적으로 동일하다. 생성 품질 평가(Track 2, 3, 5, 7)도 동일 샘플링 파라미터(temperature=0.7, repeat_penalty=1.2, top_p=0.9)를 사용하여 동일한 결과 분포를 기대할 수 있다.

### 3.2 FRANKENSTALLM v2 특성

- **실제 파라미터**: 1.2B (명칭은 "3B"이나 실제는 1.2B)
- **토크나이저**: SentencePiece Model (SPM) -- 대부분의 비교 모델이 BPE를 사용하는 것과 차이
- **학습 방법**: ORPO (Odds Ratio Preference Optimization) -- 한국어 데이터로 fine-tuning
- **v1 vs v2**: v1은 SPM 결함으로 실행 불가(CRASH), v2는 실행 가능하나 품질 미달

### 3.3 비교 모델 선정 근거

모든 비교 모델은 2~4B 파라미터 범위의 소형 모델로, 한국어를 지원하며 Ollama를 통해 추론 가능한 모델이다.

### 7.2 평가 프롬프트 샘플

- **Track 1**: KMMLU 형식의 4지선다 객관식 (예: "다음 중 올바른 것을 고르시오")
- **Track 2**: Ko-Bench 8카테고리 멀티턴 대화 (예: "한국의 전통 음식에 대해 설명해주세요" -> 후속 질문)
- **Track 3**: 한국어 심화 100문항 (존댓말/반말 전환, 사자성어, 맞춤법 등)
- **Track 4**: Python 코드 생성, SQL 쿼리 작성, 버그 수정, 수학 문제 풀이
- **Track 5**: 동일 질문 반복, 패러프레이즈, 길이 변경, 언어 전환, 구조적 출력, 환각 유도
- **Track 7**: 동일 프롬프트에 대한 2개 모델 응답을 Claude Judge가 쌍대 비교

### 7.3 환경 재현 가이드

1. Ollama 설치 및 모델 다운로드
2. `CUDA_VISIBLE_DEVICES=""` 설정으로 CPU-only 모드 활성화
3. `config.py`의 `_TIMEOUT_MULTIPLIER=2` 확인
4. 각 트랙별 평가 스크립트 실행
5. 상세 절차는 프로젝트 `README.md` 참조
